# Roadmap

## 1. Roadmap Philosophy

The project should be built in layers:

1. Foundation
2. Commerce
3. Search and content
4. AI MVP
5. ML personalization
6. Research extensions
7. AI operations readiness
8. MVP hardening

This order minimizes rework and makes the project trainable for future AI agents.

## 2. Phase 0 - Foundation

Duration:
- immediate

Goals:
- establish project direction
- lock architecture
- define data model
- define core APIs
- define implementation order

Deliverables:
- README
- architecture docs
- ERD docs
- API docs
- module requirements docs
- roadmap

## 3. Phase 1 - Storefront and Admin UI Skeleton

Goals:
- choose UX direction
- create layout system
- create reusable UI components
- create mock pages

Deliverables:
- storefront shell
- admin shell
- home page mock
- category page mock
- PDP mock
- admin dashboard mock
- seeded mock catalog for 60 products

## 4. Phase 2 - Commerce Core MVP

Goals:
- make the platform function as basic ecommerce

Deliverables:
- auth
- catalog APIs
- category browsing
- product detail pages
- cart
- checkout shell
- orders
- admin product CRUD

## 5. Phase 3 - Search and Content

Goals:
- improve findability and trust

Deliverables:
- keyword search
- filters
- article system
- FAQ system
- richer product metadata

## 6. Phase 4 - AI MVP

Goals:
- add useful AI without overcomplicating the platform

Deliverables:
- FAQ chatbot
- product Q&A assistant
- rule-based recommendations
- semantic search with pgvector
- AI session logging
- human handoff summary

## 7. Phase 5 - ML Personalization

Goals:
- turn interaction data into personalization

Deliverables:
- event ingestion
- analytics surfaces
- heuristic recommendation ranking
- embedding-based search
- personalized shelves

## 8. Phase 6 - Research and Experimentation

Goals:
- deepen academic value

Candidate work:
- collaborative filtering
- learning-to-rank
- demand forecasting
- prompt experiments
- recommendation evaluation experiments
- safety benchmarking

## 9. Phase 7 - AI Operations Readiness

Goals:
- make AI features observable and reviewable
- keep evaluation snapshots for drift checks
- expose readiness gates before moving from mock/local mode to cloud mode

Deliverables:
- AI readiness report
- evaluation history snapshots
- cloud configuration checks
- embedding coverage checks
- recommendation quality thresholds
- next action list for production hardening

## 10. Phase 8 - MVP Hardening

Goals:
- close the most visible ecommerce gaps
- make the demo flow feel complete end-to-end
- document what is still mock/local before production

Deliverables:
- order detail page
- admin order operations page
- storefront navigation on cart, checkout, and order surfaces
- MVP status document
- final typecheck and production build verification

## 11. Suggested Immediate Implementation Sequence

To begin coding after Phase 0 docs:

1. Replace dashboard shell with storefront shell
2. Define shared domain types
3. Create mock catalog dataset
4. Build homepage sections
5. Build category/search listing
6. Build PDP
7. Build cart
8. Build admin shell

Implementation note:
- semantic search should use `Gemini text-embedding-004` + `Supabase pgvector`
- product embeddings should be generated by a one-time enrichment script after seeding mock data
